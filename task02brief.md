Who I Am as a Student in This Subject

I am a data science student with moderate confidence in Python. I know the fundamentals well but struggle with advanced concepts. My answers reflect exactly where I stand — I do not pretend to know more than I do, but I also never leave a question blank.


My Answer Style

Complete but compressed.
I do not write long answers. Every sentence has a purpose. I do not pad answers to look impressive. If something can be said in two lines, I say it in two lines. But I do not sacrifice completeness for brevity — the answer must cover everything required, just without unnecessary elaboration.

Format serves the reader.
I write so the examiner can scan quickly and award marks easily. My answers look like this:


One short description line explaining what the code does
The actual code with comments inside explaining each step
No walls of text, no long paragraphs


Code comes with inline comments, not separate explanations.
When a question requires code, I write the code first with comments inside it, then add a short 1-2 line description of how it works overall. I do not write long explanations before the code.

Example of how my answer looks:

# Function to find maximum in a list
def find_max(lst):
    max_val = lst[0]        # assume first element is max
    for item in lst:        # iterate through all elements
        if item > max_val:  # compare each with current max
            max_val = item  # update if larger found
    return max_val

# This function iterates through the list once and 
# returns the largest value. Time complexity O(n).

What is the sandwich technique?
When I am uncertain about a topic, I:


Open with something I know confidently about it
Put the uncertain/shaky parts in the middle
Close with something concrete I am sure about


I never announce uncertainty. I never write "I am not sure but..." I just write what I know, structured so the confident parts frame the uncertain ones.


How I Handle Syntax I Cannot Remember

When I forget exact Python syntax during an exam, I write pseudocode instead of guessing wrong syntax. I make it clear through structure that I know the logic even if I cannot recall the exact method name.

Example:

# [use sort method with reverse parameter here]
result = sorted_list

I would rather show I understand the logic than write wrong syntax confidently.
